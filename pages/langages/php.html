Les 10 Éco-Rules PHP

1 concat2implode - Concaténation de chaînes en boucle

Impact ÉLEVÉ

Code inefficace :

$result = '';
foreach ($items as $item) {
    $result .= $item . ', ';
}
$result = rtrim($result, ', ');

Pourquoi c'est inefficace ?

  • Chaque concaténation $result .= ... crée une nouvelle chaîne en mémoire
  • Pour 1000 éléments : 1000 allocations mémoire distinctes
  • Surconsommation mémoire et temps CPU élevé

Code optimisé :

$result = implode(', ', $items);

Gains mesurés :

  • ×4 plus rapide pour les grandes listes [3]
  • Réduit la consommation mémoire de 60-80%
  • Une seule allocation mémoire au lieu de N

2 echo_concat2comma - Affichages avec virgules

Impact MOYEN

Code inefficace :

echo 'Bonjour ' . $nom . ', vous avez ' . $age . ' ans.';

Pourquoi c'est inefficace ?

  • La concaténation avec . crée une chaîne temporaire complète
  • Impact mémoire inutile pour des affichages simples

Code optimisé :

echo 'Bonjour ', $nom, ', vous avez ', $age, ' ans.';

Gains mesurés :

  • Réduit l'utilisation mémoire (pas de chaîne temporaire)
  • echo avec virgules envoie directement au buffer de sortie [4]

3 sprintf2direct - Éviter sprintf() inutile

Impact ÉLEVÉ

Code inefficace :

$message = sprintf('Utilisateur %s a %d points', $nom, $points);
echo $message;

Pourquoi c'est inefficace ?

  • sprintf() est ×3 plus lent que la concaténation directe [5]
  • Parsing du format + allocation mémoire temporaire
  • Overhead de fonction inutile pour des cas simples

Code optimisé :

// Pour affichage direct :
echo "Utilisateur $nom a $points points";

// Ou avec printf() pour éviter l'allocation temporaire :
printf('Utilisateur %s a %d points', $nom, $points);

Gains mesurés :

  • Temps d'exécution réduit de 60% (0.27s vs 0.61s pour 500k itérations) [5]
  • Moins de mémoire utilisée

4 in_array2isset - Recherche dans tableaux

Impact TRÈS ÉLEVÉ

Code inefficace :

$allowed_roles = ['admin', 'editor', 'author', 'subscriber'];

if (in_array($user_role, $allowed_roles)) {
    // Accès autorisé
}

Pourquoi c'est inefficace ?

  • in_array() parcourt séquentiellement tout le tableau (O(n))
  • Pour 10 000 éléments : ×1000 plus lent que isset() [6]
  • Temps CPU élevé pour tableaux volumineux

Code optimisé :

// Utiliser les clés comme index (O(1)) :
$allowed_roles = [
    'admin' => true,
    'editor' => true,
    'author' => true,
    'subscriber' => true
];

if (isset($allowed_roles[$user_role])) {
    // Accès autorisé
}

Gains mesurés :

  • Performance : isset() 0.003s vs in_array() 3.78s [6]
  • Complexité : O(1) au lieu de O(n)
  • Économie CPU significative sur grandes listes

5 array_push2direct - Ajout direct aux tableaux

Impact FAIBLE

Code inefficace :

$results = [];
foreach ($items as $item) {
    array_push($results, process($item));
}

Pourquoi c'est inefficace ?

  • array_push() est un appel de fonction avec overhead
  • PHP doit résoudre la fonction et gérer la pile d'appels à chaque itération

Code optimisé :

$results = [];
foreach ($items as $item) {
    $results[] = process($item);
}

Gains mesurés :

  • Évite l'overhead de l'appel de fonction
  • Code plus lisible et idiomatique
  • Performance améliorée en boucles intensives [7]

6 array_merge_loop - Fusion unique de tableaux

Impact TRÈS ÉLEVÉ

Code inefficace :

$all_data = [];
foreach ($sources as $source) {
    $all_data = array_merge($all_data, fetch_data($source));
}

Pourquoi c'est inefficace ?

  • array_merge() recrée un nouveau tableau à chaque itération
  • Pour n itérations : complexité O(n²) en mémoire et temps
  • Consommation mémoire exponentielle et CPU élevé

Code optimisé :

// PHP 7.4+ avec spread operator :
$arrays_to_merge = [];
foreach ($sources as $source) {
    $arrays_to_merge[] = fetch_data($source);
}
$all_data = array_merge(...$arrays_to_merge);

// Alternative PHP < 7.4 :
$all_data = [];
foreach ($sources as $source) {
    foreach (fetch_data($source) as $item) {
        $all_data[] = $item;
    }
}

Gains mesurés :

  • Complexité : O(n²) → O(n)
  • Évite les copies répétées en mémoire
  • Réduit drastiquement les opérations CPU [7]

7 opcache_enabled - Activer OPcache en production

Impact CRITIQUE

Configuration inefficace :

; php.ini
opcache.enable=0

Pourquoi c'est inefficace ?

  • Sans OPcache, PHP recompile les scripts à chaque requête
  • Temps de parsing/compilation répété → surconsommation CPU
  • Pour 1000 req/min : des milliers de recompilations inutiles

Configuration optimisée :

; php.ini
opcache.enable=1
opcache.memory_consumption=128
opcache.interned_strings_buffer=8
opcache.max_accelerated_files=10000
opcache.revalidate_freq=60
opcache.validate_timestamps=1  ; 0 en production stricte

Gains mesurés :

  • Performance : ×3-5 plus rapide selon benchmarks [8]
  • Réduit drastiquement la charge processeur
  • Moins de cycles CPU = moins de consommation électrique

8 foreach_reference_cleanup - Nettoyage des références

Impact MOYEN

Code dangereux :

$items = [1, 2, 3, 4, 5];

foreach ($items as &$item) {
    $item *= 2;
}

// ⚠️ La référence $item persiste !
foreach ($items as $item) {
    echo $item;  // Comportement inattendu !
}

Pourquoi c'est dangereux ?

  • La référence &$item persiste après le foreach
  • Dernière itération du 2ᵉ foreach modifie l'avant-dernier élément
  • Bugs silencieux + surconsommation mémoire (références orphelines)

Code optimisé :

$items = [1, 2, 3, 4, 5];

foreach ($items as &$item) {
    $item *= 2;
}
unset($item);  // ✅ Libère la référence

foreach ($items as $item) {
    echo $item;  // Comportement correct
}

Gains mesurés :

  • Évite les bugs de mutation involontaire
  • Libère la mémoire des références orphelines
  • Meilleure prévisibilité du code [9]

9 empty2isset_check - Vérifications ciblées

Impact FAIBLE

Code inefficace :

$config = load_config();

if (!empty($config['database']['host'])) {
    connect($config['database']['host']);
}

Pourquoi c'est inefficace ?

  • empty() effectue plusieurs vérifications (existence + falsiness)
  • Overhead inutile si la variable existe toujours

Code optimisé :

$config = load_config();

if (isset($config['database']['host']) && $config['database']['host']) {
    connect($config['database']['host']);
}

Gains mesurés :

  • if (!$a) est 8-10% plus rapide que !empty($a) [10]
  • Moins d'overhead de vérification

10 require2autoload - Autoloading PSR-4

Impact MOYEN

Code inefficace :

// Chargement manuel de toutes les classes
require_once 'models/User.php';
require_once 'models/Product.php';
require_once 'models/Order.php';
require_once 'controllers/UserController.php';
// ... 50 autres fichiers

Pourquoi c'est inefficace ?

  • Charge tous les fichiers même ceux non utilisés
  • Surconsommation mémoire et temps de parsing
  • Maintenance difficile (ajouter require pour chaque nouvelle classe)

Code optimisé :

// Autoloader PSR-4 (ou utiliser Composer)
spl_autoload_register(function ($class) {
    $file = str_replace('\\', DIRECTORY_SEPARATOR, $class) . '.php';
    if (file_exists($file)) {
        require $file;
    }
});

// Les classes sont chargées automatiquement à la demande
$user = new User();  // Charge User.php seulement si nécessaire

Gains mesurés :

  • Chargement à la demande → moins de fichiers en mémoire
  • Évite le parsing de code inutilisé
  • Meilleure organisation et maintenabilité [11]

Bonnes pratiques complémentaires

Types stricts

declare(strict_types=1);

function calculateTotal(float $price, int $qty): float {
    return $price * $qty;
}

Évite les conversions de types implicites → économie CPU

Requêtes SQL

// ✅ Une seule requête
$ids = implode(',', $user_ids);
$users = $db->query(
    "SELECT * FROM users WHERE id IN ($ids)"
);

Évite le problème N+1 → moins de connexions réseau

Générateurs

function getAllRecords($pdo) {
    $stmt = $pdo->query("SELECT * ...");
    while ($row = $stmt->fetch()) {
        yield $row;
    }
}

Mémoire constante au lieu de charger 100k lignes
