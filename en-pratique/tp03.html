TP 03 : Détection et Réduction des Scripts Inutilisés

Objectif +

Dans ce TP, vous allez identifier et éliminer les scripts JavaScript inutilisés ou redondants sur le site web du Bureau des Étudiants (BDE) de l'ESIROI. L'objectif est de réduire la taille des fichiers JavaScript, d'améliorer les performances du site, et de minimiser son empreinte écologique.

Temps imparti : 1 heure

Date et lieux de dépôt : À déterminer

Informations +

Les scripts JavaScript non utilisés ou redondants peuvent entraîner des temps de chargement plus longs, consommer des ressources inutiles, et augmenter l'empreinte écologique d'un site web. Ce TP vous apprendra à identifier ces scripts et à les supprimer ou les optimiser pour rendre le site plus performant et éco-responsable.

Prérequis nécessaires +

  • Connaissances de base en HTML, CSS, et JavaScript
  • Docker et Docker Compose installés sur votre machine
  • Accès à une machine avec Ubuntu, Windows, ou macOS
  • Installation de SonarQube et SonarScanner (voir TP 01)
  • Analyse initiale du site réalisée dans le TP 02

Ressources utiles +

  • Chrome DevTools
  • Firefox Developer Tools
  • SonarQube

Livrable +

À la fin de ce TP, vous devrez compléter une section de votre rapport individuel décrivant les optimisations effectuées, les résultats avant et après optimisation, et une évaluation de l'impact des optimisations sur les performances et l'empreinte écologique du site. Chaque TP constituera une partie distincte de ce rapport, qui sera complété au fur et à mesure des travaux pratiques.

Rappel des critères du rapport :

  • Longueur : 10 pages maximum
  • Organisation du rapport :
    • Introduction : Présentation générale du projet et de ses objectifs.
    • Section 1 : Installation et Configuration de SonarQube et Eco Code
    • Section 2 : Optimisation des Ressources Web
    • Section 3 : Détection et Réduction des Scripts Inutilisés
    • Section 4 : Optimisation de l'Utilisation des Polices Web
    • Conclusion : Résumez les apprentissages et les recommandations principales

Étapes du TP +

  1. Étape 1 : Analyse initiale des scripts

    1. Clonage du dépôt GitHub: Si ce n'est pas déjà fait, clonez le dépôt contenant le site vitrine du BDE.
      2. git clone https://github.com/username/bde-esiroi.git
    2. Déplacez-vous dans le répertoire du projet :
      3. cd bde-esiroi
    3. Audit des scripts avec Chrome DevTools :
      • Ouvrez le site dans Google Chrome.
      • Accédez à Chrome DevTools en appuyant sur F12 ou Ctrl + Shift + I.
      • Allez dans l'onglet Sources puis dans Coverage.
      • Cliquez sur Start Instrumenting Coverage pour démarrer l'analyse des scripts.
      • Naviguez sur les différentes pages du site pour capturer l'utilisation des scripts.
      • Analysez le rapport généré pour identifier les pourcentages de code JavaScript réellement utilisé.
      • Prenez des captures d'écran des résultats pour les inclure dans votre rapport.

  2. Étape 2 : Suppression ou optimisation des scripts inutilisés

    1. Suppression des scripts inutilisés :
      • Supprimez les fichiers JavaScript qui ne sont pas utilisés du projet.
      • Si des portions de code non utilisées sont trouvées dans des fichiers JS encore nécessaires, supprimez uniquement ces portions tout en testant pour éviter les régressions.
    2. Optimisation des scripts :
      • Refactorisez les scripts qui sont partiellement utilisés pour les rendre plus efficaces.
      • Si des bibliothèques ou des frameworks JavaScript sont utilisés, envisagez de les remplacer par des alternatives plus légères si elles sont surdimensionnées pour le projet.
    3. Test du site :
      • Testez le site après chaque modification pour vous assurer que la suppression ou l'optimisation des scripts n'a pas causé de dysfonctionnements.
      • Utilisez les outils de développement pour vérifier que les scripts restants sont effectivement nécessaires.

  3. Étape 3 : Analyse post-optimisation

    1. Réalisation d'une nouvelle analyse avec Chrome DevTools :
      • Refaites l'analyse de couverture des scripts avec Chrome DevTools après optimisation.
      • Comparez les nouveaux résultats avec ceux obtenus avant optimisation.
    2. Analyse avec SonarQube:
      • Effectuez une nouvelle analyse du site avec SonarQube pour détecter d'éventuels autres problèmes liés aux scripts.
      • Documentez les recommandations supplémentaires fournies par SonarQube.

Conclusion: Rédaction du rapport +

Documentez les résultats de l'analyse et proposez des recommandations spécifiques pour améliorer le site. Estimez l'impact de ces améliorations sur les performances du site et sur son empreinte écologique.
