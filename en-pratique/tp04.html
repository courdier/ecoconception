TP 04 : Optimisation de l'Utilisation des Polices Web

Objectif +

Dans ce TP, vous allez optimiser l'utilisation des polices web sur le site du Bureau des Étudiants (BDE) de l'ESIROI. L'objectif est de réduire la charge des ressources liées aux polices, d'améliorer les performances du site, et de minimiser son empreinte écologique.

Temps imparti : 2 heures

Date et lieux de dépôt : À déterminer

Informations +

L'utilisation inefficace des polices web peut ralentir le chargement des pages et augmenter la consommation de ressources, ce qui impacte à la fois la performance et l'empreinte écologique du site. Ce TP vous apprendra à choisir et à intégrer des polices web de manière optimisée, en utilisant des techniques telles que le sous-ensemble de polices, la mise en cache et l'utilisation de formats de polices optimisés.

Prérequis nécessaires +

  • Connaissances de base en HTML, CSS, et JavaScript
  • Docker et Docker Compose installés sur votre machine
  • Accès à une machine avec Ubuntu, Windows, ou macOS
  • Installation de SonarQube et SonarScanner (voir TP 01)
  • Analyse initiale du site réalisée dans les TP précédents

Ressources utiles +

  • Google Fonts
  • WOFF2 Format
  • Webfont Loader
  • Google Fonts Subsetter

Livrable +

À la fin de ce TP, vous devrez compléter une section de votre rapport individuel décrivant les optimisations effectuées, les résultats avant et après optimisation, et une évaluation de l'impact des optimisations sur les performances et l'empreinte écologique du site. Chaque TP constituera une partie distincte de ce rapport, qui sera complété au fur et à mesure des travaux pratiques.

Rappel des critères du rapport :

  • Longueur : 10 pages maximum
  • Organisation du rapport :
    • Introduction : Présentation générale du projet et de ses objectifs.
    • Section 1 : Installation et Configuration de SonarQube et Eco Code
    • Section 2 : Optimisation des Ressources Web
    • Section 3 : Détection et Réduction des Scripts Inutilisés
    • Section 4 : Optimisation de l'Utilisation des Polices Web
    • Conclusion : Résumez les apprentissages et les recommandations principales

Étapes du TP +

  1. Étape 1 : Analyse initiale de l'utilisation des polices web

    1. Clonage du dépôt GitHub : Si ce n'est pas déjà fait, clonez le dépôt contenant le site vitrine du BDE.
      2. git clone https://github.com/username/bde-esiroi.git
    2. Déplacez-vous dans le répertoire du projet :
      3. cd bde-esiroi
    3. Identification des polices utilisées : Inspectez les fichiers CSS et HTML pour identifier les polices web utilisées sur le site. Notez les polices importées via des services externes (ex : Google Fonts) et celles chargées localement.
    4. Analyse de l'impact des polices sur la performance : Utilisez Google Lighthouse ou WebPageTest pour évaluer l'impact des polices sur les performances du site (ex : temps de chargement, poids des fichiers). Prenez des captures d'écran des résultats pour les inclure dans votre rapport.

  2. Étape 2 : Optimisation des polices web

    1. Sélection des polices nécessaires : Analysez les polices utilisées pour déterminer si toutes sont nécessaires. Supprimez les polices superflues ou redondantes. Si plusieurs styles ou graisses (font-weight) d'une même police sont chargés, évaluez s'il est possible de réduire le nombre de variations.
    2. Sous-ensemble de polices (Font Subsetting) : Créez un sous-ensemble (subset) des polices pour inclure uniquement les caractères nécessaires au site. Utilisez des outils comme Google Fonts Subsetter pour générer un sous-ensemble de la police.
    3. Utilisation de formats de polices optimisés : Convertissez les polices au format WOFF2 si ce n'est pas déjà fait, car ce format est plus léger et performant. Assurez-vous que les polices WOFF2 sont bien servies en priorité par rapport aux autres formats (ex : TTF, OTF).
    4. Mise en cache des polices : Configurez le serveur pour mettre en cache les fichiers de polices pour une période prolongée. Si vous utilisez un CDN, vérifiez que la mise en cache est correctement configurée pour les polices.

  3. Étape 3 : Analyse post-optimisation

    1. Réalisation d'une nouvelle analyse avec Google Lighthouse : Refaites l'analyse des performances du site après avoir appliqué les optimisations des polices. Comparez les nouveaux résultats avec ceux obtenus avant optimisation.
    2. Analyse avec SonarQube : Effectuez une nouvelle analyse du site avec SonarQube pour vérifier que les modifications n'ont pas introduit de nouvelles problématiques en termes de performance ou d'éco-conception. Documentez les recommandations supplémentaires fournies par SonarQube.

Conclusion: Rédaction du rapport +

Documentez les résultats de l'analyse et proposez des recommandations spécifiques pour améliorer le site. Estimez l'impact de ces améliorations sur les performances du site et sur son empreinte écologique.
