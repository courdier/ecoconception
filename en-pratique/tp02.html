TP 02 : Optimisation des Ressources Web

Objectif +

Dans ce TP, vous allez optimiser les ressources du site web du Bureau des Étudiants (BDE) de l'ESIROI afin de réduire son empreinte écologique et d'améliorer ses performances. L'objectif est de mettre en œuvre différentes techniques d'optimisation, notamment la compression d'images, la minification et la concaténation des fichiers CSS et JavaScript, ainsi que l'utilisation de la mise en cache.

Temps imparti : 2 heures

Date et lieux de dépôt : À déterminer

Informations +

L'optimisation des ressources web est essentielle pour améliorer les performances d'un site tout en réduisant la consommation de bande passante et l'empreinte écologique. Dans ce TP, vous allez apprendre à appliquer ces techniques sur le site web du BDE, ce qui permettra de réduire le temps de chargement des pages et de minimiser l'impact environnemental du site.

Prérequis nécessaires +

  • Connaissances de base en HTML, CSS, et JavaScript
  • Docker et Docker Compose installés sur votre machine
  • Accès à une machine avec Ubuntu, Windows, ou macOS
  • Installation de SonarQube et SonarScanner (voir TP 01)

Ressources utiles +

  • Google Lighthouse
  • TinyPNG
  • ImageOptim
  • UglifyJS
  • cssnano

Livrable +

À la fin de ce TP, vous devrez compléter une section de votre rapport individuel décrivant les optimisations effectuées, les résultats avant et après optimisation, et une évaluation de l'impact des optimisations sur les performances et l'empreinte écologique du site. Chaque TP constituera une partie distincte de ce rapport, qui sera complété au fur et à mesure des travaux pratiques.

Rappel des critères du rapport :

  • Longueur : 10 pages maximum
  • Organisation du rapport :
    • Introduction : Présentation générale du projet et de ses objectifs.
    • Section 1 : Installation et Configuration de SonarQube et Eco Code
    • Section 2 : Optimisation des Ressources Web
    • Section 3 : Détection et Réduction des Scripts Inutilisés
    • Section 4 : Optimisation de l'Utilisation des Polices Web
    • Conclusion : Résumez les apprentissages et les recommandations principales

Étapes du TP +

  1. Étape 1 : Analyse initiale des performances

    1. Clonage du dépôt GitHub :Si ce n'est pas déjà fait, clonez le dépôt contenant le site vitrine du BDE.
      2. git clone https://github.com/username/bde-esiroi.git
    2. Déplacez-vous dans le répertoire du projet :
      3. cd bde-esiroi
    3. Analyse avec Google Lighthouse : Ouvrez le projet dans un navigateur et utilisez Google Lighthouse pour analyser les performances du site.
      • Notez les scores pour les catégories suivantes : Performance, Accessibilité, Bonnes pratiques, SEO, Éco-conception (si disponible via des plugins supplémentaires).
      • Capturez les résultats pour les comparer après optimisation.

  2. Étape 2 : Optimisation des images

    1. Analyse des images existantes : Identifiez les images les plus volumineuses du site et analysez leur taille, format, et impact sur le temps de chargement.
    2. Compression des images : Utilisez des outils comme TinyPNG ou ImageOptim pour compresser les images sans perte de qualité visible. Remplacez les images originales par les versions compressées dans le projet.
    3. Vérification de la qualité : Assurez-vous que la qualité des images reste acceptable après compression. Testez le site pour vérifier que les images optimisées s'affichent correctement.

  3. Étape 3 : Minification et Concatenation des fichiers CSS et JS

    1. Minification des fichiers CSS : Utilisez un outil comme cssnano pour minifier les fichiers CSS du projet. Remplacez les fichiers CSS originaux par les versions minifiées.
    2. Minification des fichiers JavaScript : Utilisez UglifyJS pour minifier les fichiers JavaScript du projet. Remplacez les fichiers JS originaux par les versions minifiées.
    3. Concatenation des fichiers : Combinez les fichiers CSS et JavaScript pour réduire le nombre de requêtes HTTP. Modifiez les références dans le code HTML pour pointer vers les fichiers concaténés.

  4. Étape 4 : Mise en cache

    1. Configuration du cache : Configurez la mise en cache des ressources statiques (images, CSS, JS) pour que le navigateur les conserve plus longtemps. Si vous utilisez un serveur web local, modifiez les paramètres de configuration pour activer la mise en cache.
    2. Test des paramètres de cache : Testez le site web avec Google Lighthouse pour vérifier si la mise en cache fonctionne correctement. Notez les améliorations apportées par cette optimisation.

Conclusion: Rédaction du rapport +

Documentez les résultats de l'analyse et proposez des recommandations spécifiques pour améliorer le site. Estimez l'impact de ces améliorations sur les performances du site et sur son empreinte écologique.
